auth.model.js for verification of user using tokens:-
JWT Verification Flow

1. Secret Key Setup
- JWT_SECRET_KEY is stored in .env.
- Used during both token creation (jwt.sign) and token verification (jwt.verify).

2. Token Creation (Login)
const token = jwt.sign(
  { id: user._id },
  process.env.JWT_SECRET_KEY,
  { expiresIn: "1h" }
);

- Payload contains id: user._id.
- This payload is embedded inside the JWT.

3. Token Arrival
- Server sends token to browser (usually in a cookie).
- Browser automatically sends the token with future requests.

4. Token Extraction
const token = req.cookies?.token;

- Reads the JWT sent by the browser.

5. Token Verification
const encoded = jwt.verify(token, process.env.JWT_SECRET_KEY);

- Verifies token signature using the secret key.
- Checks token has not been tampered with.
- Checks token has not expired.
- Returns decoded payload.

Example encoded:
{
  id: "userId123",
  iat: ...,
  exp: ...
}

6. User Lookup
const user = await userModel.findById(encoded.id);

- encoded.id comes from the payload defined during jwt.sign().
- Finds the corresponding user in the database.
- If user does not exist, return error.

7. Attach User
req.user = user;

- Makes authenticated user data available to subsequent middleware/routes.

8. Continue Request
next();

- Passes control to the next middleware or route handler.

Key Idea:
- Token stores user information defined during jwt.sign().
- jwt.verify() validates the token and returns that stored information.
- encoded.id is the same id that was originally placed in the payload during token creation.
